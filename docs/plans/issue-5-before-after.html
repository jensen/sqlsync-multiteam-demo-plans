Before/After: FormData Type Safety
❌ BEFORE
const body = Object.fromEntries( await request.formData() ) as unknown as { title: string; priority: string; team: string; }; // No runtime validation! // File uploads pass through silently // Missing fields become undefined // Type safety is a lie at runtime

Problems

  • Casts bypass all validation
  • Files treated as strings
  • Missing fields → undefined
  • Silent data corruption
→
✅ AFTER
const body = parseFormData<{ title: string; priority: string; team: string; }>(await request.formData(), [ "title", "priority", "team" ]); // Runtime validation guarantees: // - All fields are present // - All values are strings // - Errors thrown immediately

Benefits

  • Every field validated at runtime
  • Files rejected with clear errors
  • Missing fields throw immediately
  • Type safety is real, not fiction
